# 07 — Full-code audit and desk-test round 5 fixes (2026-07-22)

An end-to-end read of the Flutter frontend and its bridge, prompted by the
collaborator's round-5 desk test. Three defects were confirmed, root-caused and
fixed on this branch; this note records what changed, how to find your way
around the affected machinery, and what remains open. The blow-by-blow defect
entries live in `docs/test-feedback-log.md` (round 5); this is the map.

## The three fixes, in one paragraph each

**The preview now follows edits** (`eecd472`). The engine's rendered-frame cache
was always invalidated correctly on an edit — the stale picture came from the
*Dart-side* image cache in `flutter_ui/lib/panels/preview_source.dart`, which
keyed frames by comp + frame + scale but not by document version. It now tracks
`AppStateStub.documentEpoch`: an edit drops the cache, the epoch rides every
cache key, and a render reply that lands after an edit is discarded instead of
banked. Scrubbing still hits the cache (playhead motion never bumps the epoch).

**The interface no longer freezes on uncached renders** (`54d2f68`). The comp
render was already off the interface thread; the freeze came from four other
engine calls still made *on* it, each of which had to wait for the engine's
renderer lock while a render was in flight: the Scopes trace (the systematic
one — it fired on every landed frame), Project-panel thumbnails, Detect beats
(which blocked for the whole audio analysis) and the eyedropper's frame sample.
All four now run off the interface thread — the first two through the existing
render worker, beats in its own short-lived worker, the eyedropper by sampling
the frame the Viewer already holds.

**Audio playback exists** (`bcc585a`). Play used to move the picture silently;
nothing ever opened the engine's audio output. The bridge grew an audio surface
(v0.10, ABI 10): prepare/play/pause/seek/stop plus a cheap per-tick clock poll,
and the Viewer's transport now treats the sound card's clock as the playback
master — the picture chases it, exactly as the egui frontend and the docs
(docs/09, docs/impl/playback-scheduler.md §4) prescribe. Edits while playing
re-prepare the mix; an unchanged mix is a no-op and a changed one is heard in
about ten milliseconds without losing the clock. A machine with no audio
device, a silent comp, or an old library falls back to the wall clock — the
old behaviour, unchanged.

## Where things live now (the navigation guide)

| Concern | File(s) | What to know before changing it |
|---|---|---|
| Which picture the Viewer shows, and the Dart image cache | `flutter_ui/lib/panels/preview_source.dart` | Three paths tried in order: shared GPU texture → read-back comp render → single-layer decode. At most one render in flight (latest-wins). Cache keys carry the document epoch; if you add a new cached thing, fold the epoch in or it will serve stale pixels after an edit. |
| The render worker (everything heavy, off the interface thread) | `flutter_ui/lib/panels/preview_isolate.dart` | One long-lived worker isolate opens its own handle to the same engine library. Request kinds: `comp`, `decode`, `shared`, `scope`, `thumb`. To move another engine call off-thread, add a kind here and a `request…` method on the `FrameRenderer` seam — the `SynchronousFrameRenderer` twin keeps widget tests deterministic. |
| Scopes | `flutter_ui/lib/panels/scopes_panel.dart` | Traces are request→callback now, never synchronous. The CPU fallback stays for old libraries. |
| Thumbnails | `flutter_ui/lib/panels/project_panel.dart` | Decoded through the worker via the app-state seam; the epoch guard re-decodes after a relink. |
| Beat detection | `flutter_ui/lib/state/app_state.dart` (`detectBeats`), `flutter_ui/lib/bridge/beats_worker.dart` | Runs in a short-lived isolate with its own library handle; the reply is adopted on the interface thread through the normal op path. Fake bridges keep the synchronous path. |
| Audio playback, Dart side | `flutter_ui/lib/bridge/bridge.dart` (`AudioPlaybackBridge`), `flutter_ui/lib/state/app_state.dart` (play/pause/seek wiring), `flutter_ui/lib/panels/viewer_panel.dart` (`_onTick` chases the clock) | The ticker polls the clock once per tick. If you change transport semantics (what stops vs pauses), keep the audio calls in step — the tests in `test/audio_playback_test.dart` pin the pairs. |
| Audio playback, engine side | `crates/lumit-bridge/src/audio.rs`, `crates/lumit-audio/src/lib.rs` (`ClockHandle`), `crates/lumit-ui/src/headless.rs` (`AudioJobsBuilder`) | The audio engine lives on its own thread (the cpal stream cannot move between threads); commands go over a channel. Preparing a mix never touches the GPU renderer or its lock. The jobs signature decides no-op vs swap — if you add a model field that changes what a comp *sounds* like, add it to the signature or edits won't be heard. |
| The bridge ABI | `crates/lumit-bridge/src/lib.rs` (`ABI_VERSION` = 10), `crates/lumit-bridge/src/ffi.rs` | Every new export must degrade calmly when absent (symbol-presence gating on the Dart side), so an old dll never crashes the app. |

## What to rebuild after pulling this branch

Both halves changed, so both need rebuilding: the engine library (`cargo build
-p lumit-bridge --release`, plus `--features shared-texture` on Windows for the
zero-copy Viewer) and the Flutter app (`flutter build windows` / `flutter build
linux` in `flutter_ui/`). An old dll under a new app runs — the new surfaces
gate on symbol presence — but audio and the off-thread scope/thumbnail paths
only light up with the new library.

## Known open items (recorded, not fixed here)

- **Pop-out: the floating panel's tab stays in the main window** showing a
  placeholder. The owner logged this round 4 as "a later issue"; the
  hide-while-floating behaviour (the egui contract) is still the intent. See
  `flutter_ui/lib/shell/shell.dart` `_floating`.
- **Cross-window drag from a popped-out panel** cannot work engine-to-engine;
  a bridge-mediated hand-off is the recorded follow-up (round 4).
- **Cached-mode audio gating** (K-171): the Flutter picture free-runs during
  playback, so sound can lead an uncached stretch where egui would pause it.
  Named in `06-REMAINING-WORK.md`.
- **Output-latency compensation** for the audio clock — within the ±half-frame
  tolerance docs/09 allows; egui omits it identically.
- **Footage-item preview audio** — comp playback only, for now.
- **Undo granularity on value drags**: a `DragValueField` drag commits an op
  per tick, so one long drag is many undo steps (egui coalesces). A
  drag-session coalescing pass is a good small follow-up.
- **The Dart image LRU holds 8 frames** (`preview_source.dart`). Engine cache
  hits keep re-scrubbing cheap regardless, but a larger blit-ready window
  (~24–32) may feel smoother on long back-and-forth scrubs; measure first.
- **Beat detection remains synchronous engine-side** (bridge caveat A in
  `06-REMAINING-WORK.md`) — it no longer blocks the interface, but a start/poll
  pair would let a progress readout exist.
